## 4. Cache POS — Stale-While-Revalidate

**Archivo:** `src/features/pos/hooks/usePosProducts.ts`

**Problema:** Al editar un producto en admin y navegar al POS, el dato desactualizado seguía mostrándose aunque se recargara con F5 (sessionStorage no se limpia con F5).

**Solución:** Patrón stale-while-revalidate:
- Si hay cache: muestra datos inmediatamente y fetchea en background.
- Si no hay cache: muestra loading y fetchea.
- El cache se actualiza siempre después de cada fetch.
- `CACHE_TTL_MS = 5 minutos`.

## 8. Bug fix — Motor de combos con misma categoría/tamaño

**Archivo:** `src/lib/promotions.ts` — función `applyCombo`

**Problema:** Al agregar dos combos que comparten el mismo tipo de ítem (ej. Pizza Pequeña), el segundo combo "robaba" la pizza ya asignada al primero, dejando un combo sin ese ítem.

**Causa:** `applyCombo` se ejecuta iterativamente por combo. Al procesar el segundo combo, los items ya habían sido modificados por el primero pero el check `qty - claimed < 1` no era suficiente porque `claimedPerIndex` se reinicia en cada llamada.

**Fix:** En el loop de matching, se skipean items que ya tienen `promo_label` asignado (ya pertenecen a otro combo). Si el item tiene `qty > 1`, el primer combo lo splitea en dos: una slice con `promo_label` y un remainder sin él — el segundo combo toma el remainder correctamente.

```ts
// Skip items already fully consumed by a previous combo
if (items[i].promo_label) continue;
```
