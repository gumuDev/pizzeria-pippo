# .claude/skills/feature/SKILL.md

1. Leer requisitos completos antes de escribir código
2. Crear el plan de implementación paso a paso antes de escribir código
3. Guardar en el folder docs/plans/ con un nombre descriptivo (ej: `2024-06-15-new-feature.md`)
4. Implementar UN módulo/componente a la vez
5. Después de cada componente, ejecutar `tsc --noEmit` y corregir errores
6. Pedir validación del usuario antes de continuar
7. Para tablas Supabase: siempre crear políticas RLS
8. Para recursos Refine: verificar nombres exactos de API para la versión instalada
